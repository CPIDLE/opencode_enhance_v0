## 觀察與心得

### OpenCode 優點
- 開源、支援多 provider（Ollama 本地、Google Gemini 雲端）
- 有 plugin/skill 系統可客製化
- AGENTS.md 類似 CLAUDE.md，可以設定 agent 行為
- subagent 模式（reviewer、explorer）很有用

### OpenCode 限制 / 問題
- 使用 Ollama 本地模型時，coding 品質不如 Claude Code
- 曾把 parser 改壞（IndentationError、import 位置錯誤、新增不存在的 class）
- 需要非常詳細的修復指示才能正確修復
- `# ... (原有內容)` 佔位問題 — 必須在 AGENTS.md 明確禁止

### 最佳使用策略
- Claude Code 為主力，OpenCode 為輔助
- OpenCode 適合：簡單任務、Claude Code 限流時的備案、節省 API 用量
- 複雜任務（多檔案重構、架構變更）仍交給 Claude Code
- 必須在 AGENTS.md 設定嚴格的修改規則和測試要求
